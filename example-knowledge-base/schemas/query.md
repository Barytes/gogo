# Query Schema

当需要从知识库回答问题时，使用这个 schema。

## 目标

优先从本地知识库回答，并在合适时把有价值的结果写回 `wiki/`。

## 默认流程

1. 先读 `wiki/index.md`
2. 优先读 `wiki/`，再读 `raw/`
3. 如果 wiki 不够，再检查 `raw/` 中的相关文件
4. 如果答案以后还会有用，就写回 `wiki/`
5. 对于有意义的写回，在 `wiki/log.md` 中追加一条简短记录

## 规则

- 对知识库相关问题，默认采用 local-only 行为
- 除非用户明确要求，否则不要浏览网页
- 回答 local-only 问题时，先列出实际查阅过的本地文件
- 如果本地材料不够，就明确说明缺什么，不要假装有把握
