# Ingest Schema

当需要把新文件处理进知识库时，使用这个 schema。

## 目标

把新资料整理成可维护的 wiki 页面。

## 默认流程

1. 读取 `inbox/` 中的新文件
2. 将文件移动或复制到 `raw/` 作为长期保存
3. 如果已有合适页面，优先更新 `wiki/` 中的已有页面
4. 只有在主题明显可复用时，才创建新 wiki 页面
5. 如果 wiki 结构发生变化，更新 `wiki/index.md`
6. 在 `wiki/log.md` 里追加一条简短记录

## 规则

- 优先更新，不要制造重复页面
- 保持 wiki 页面简洁，并补上必要链接
- 把 `raw/` 当作原始资料层
- 把可复用的总结和答案沉淀到 `wiki/`
